## Changes affecting aix

- Zed 1.4.2 introduced native Agent Skills, replacing the Rules Library.
   - Skills live at `<worktree>/.agents/skills/<name>/` (project) and `~/.agents/skills/<name>/` (user/global).
   - Each skill is a folder containing a `SKILL.md` with YAML frontmatter (`name`, `description`,
     `disable-model-invocation`) followed by a markdown instructions body.
   - The path `~/.agents/skills/` is intentionally shared with Claude Code and Codex (PR #56456).
   - Flat layout only: skills must be direct children of the root (`group/skill/` is not discovered).
   - Live reload: adding, removing, or editing `SKILL.md` takes effect immediately.
   - aix status: switch from `PointerSkillsStrategy` to `NativeSkillsStrategy` with
     `editorSkillsDir: '.agents/skills'`. aix copies skills to `.aix/skills/` and symlinks into
     `.agents/skills/<name>`. Symlinks are explicitly supported.
- Zed 1.4.2 removed the Rules Library UI entirely.
   - The old `@rule` autocomplete becomes `@skill`; slash commands invoke skills as `/skill-name`.
   - Non-default Rules Library entries migrate to `~/.agents/skills/` with
     `disable-model-invocation: true`. Default entries migrate to `~/.config/zed/AGENTS.md`.
   - aix status: pointer rules are no longer the right approach for skills. The `PointerSkillsStrategy`
     usage in the Zed adapter is removed.
- Zed 1.4.2 added `~/.config/zed/AGENTS.md` as the user-scope global rules file.
   - Windows path: `%APPDATA%\Zed\AGENTS.md`.
   - aix status: `ZedRulesStrategy.getGlobalRulesPath()` now returns `.config/zed/AGENTS.md`. User-scope
     rules install to `~/.config/zed/AGENTS.md` instead of being dropped with a limitation warning.

## Baseline evidence

- `packages/core/src/editors/strategies/zed/rules.ts` owns Zed rule output.
- `packages/core/src/editors/strategies/zed/prompts.ts` owns Zed prompt output.
- `packages/core/src/editors/strategies/zed/mcp.ts` owns Zed MCP output.
- `packages/core/src/editors/adapters/zed.ts` is the Zed adapter.
- `packages/core/src/editors/strategies/shared/native-skills.ts` is the `NativeSkillsStrategy`.
- `packages/schema/src/editor-support.ts` documents Zed's current aix feature support.
