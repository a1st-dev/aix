## Changes affecting aix

- Claude Code 2.1.147 added the `Workflow` tool behind `CLAUDE_CODE_WORKFLOWS=1`.
   - aix status: needs follow-up only if aix decides to model Claude workflow files or workflow-specific permissions. Current aix support is focused on rules, prompts, MCP, skills, agents, and hooks.
- Claude Code 2.1.145 added `background_tasks` and `session_crons` fields to `Stop` and `SubagentStop` hook input.
   - aix status: no schema change needed for hook input payloads because aix stores hook config, not runtime payload types. Docs should mention these fields if the hook guide starts documenting event payload examples.
- Claude Code 2.1.147 fixed hook `if` conditions such as `PowerShell(git push*)`.
   - aix status: already exposes the Claude Code `if` action field in `packages/schema/src/hooks.ts` and passes it through in `packages/core/src/editors/strategies/claude-code/hooks.ts`.
- Claude Code 2.1.147 fixed MCP pagination for servers, resources, templates, and prompts.
   - aix status: no install-format change needed. This affects runtime Claude Code behavior after aix writes MCP config.
- Claude Code 2.1.147 renamed `/simplify` to `/code-review`.
   - aix status: no aix implementation change. aix prompt and skill names are user-defined and do not depend on Claude Code built-in slash command names.

## Baseline evidence

- `packages/core/src/editors/strategies/claude-code/hooks.ts` maps aix hook events to Claude Code native events and passes through `if`, `async`, `asyncRewake`, HTTP hooks, MCP tool hooks, prompt hooks, and agent hooks.
- `packages/schema/src/hooks.ts` defines the cross-editor hook action fields used by the Claude Code strategy.
- `packages/schema/src/editor-support.ts` documents Claude Code support for hooks, MCP, skills, agents, prompts, and rules.
