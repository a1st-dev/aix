## Changes affecting aix

- Codex 0.133.0 enables goals by default and stores goal progress across active turns.
   - aix status: no current code change. aix does not model Codex goal storage yet.
- Codex 0.133.0 expanded permission profiles with list APIs, inheritance, managed `requirements.toml` support, runtime refresh behavior, and stronger Windows sandbox integration.
   - aix status: needs follow-up research before implementation. Current aix Codex support focuses on AGENTS.md, MCP, skills, prompts-as-skills, and rules; permission profile files are not modeled in `ai.json`.
   - addressed in aix: no change needed in this pass. Permission profiles are a Codex runtime security surface, not an existing portable aix feature. `packages/schema/src/editor-support.ts` now documents the Codex hook support found in the same research pass, but no new permission-profile schema was added.
- Codex 0.133.0 made plugin discovery easier to inspect and added visible installed versions and marketplace roots.
   - aix status: no current code change. aix does not install Codex plugins.
- Codex 0.133.0 added more extension lifecycle events, including subagent start/stop, tool execution, turn metadata, and async approval/turn processing.
   - aix status: needs follow-up if Codex exposes these events through a stable config file. `packages/schema/src/editor-support.ts` currently marks Codex hooks as unsupported.
   - addressed in aix: implemented stable Codex hook output for the documented `hooks.json` surface in `packages/core/src/editors/strategies/codex/hooks.ts`, wired it through `packages/core/src/editors/adapters/codex.ts`, and updated the support matrix in `packages/schema/src/editor-support.ts`. Current Codex docs only describe command handlers for `SessionStart`, `PreToolUse`, `PermissionRequest`, `PostToolUse`, `UserPromptSubmit`, and `Stop`, so broader extension-only lifecycle events remain unsupported.
- Codex 0.133.0 made AGENTS instruction loading more reliable, including local global reads and invalid UTF-8 warnings.
   - aix status: no format change needed. aix already treats `AGENTS.md` as the Codex instruction surface.

## Baseline evidence

- `packages/core/src/editors/strategies/codex/rules.ts` writes managed `AGENTS.md` sections for Codex rules.
- `packages/core/src/editors/strategies/codex/mcp.ts` writes Codex MCP config.
- `packages/core/src/editors/strategies/codex/prompts.ts` converts prompts into skills for Codex.
- `packages/core/src/editors/strategies/codex/hooks.ts` maps supported aix hook events into Codex `hooks.json`.
- `packages/schema/src/editor-support.ts` marks Codex hooks as native for the documented command-hook events.
